Gitlet

A version-control system that mimics some of the basic features of the popular system Git.

Unlike other class projects, Gitlet required that each student design their own implementation and build it from scratch—no skeleton code. My implementation utilized file persistence, cryptographic hashing, regex, a variety of data structures, and graph traversals. Beyond the techincal intricacies, this project emphasized software engineering design principles such as efficiency, decomposition, and clean style. Unfortunately, the design doc and code cannot be publicly shared as per course guidelines.

  • Organization: UC Berkeley CS61B (Data Structures)
  • Tech: Java
  • Link: Spec
  • Date: April 2020

uMaps

An innovative indoor mapping iOS app designed to be fully integrated with Google Maps.

0's represent walkable areas, 1's walls, and 9's the computed path.

uMaps tackled an unsolved problem in the geospatial realm: indoor navigation. While other team members worked with Swift and Google Maps API, I focused on the backend. I implemented an object-oriented approach to representing building data and used the A* algorithm to efficiently find paths. Additional functionality included accessbility features and multi-floor capability.

  • Organization: TreeHacks
  • Tech: Python • FirebaseDB
  • Link: Presentation, Repo
  • Date: February 2020

Lines of Action

A board game invented by Claude Soucie.

Aside from the implementing the game logic, each student was also required to create their own AI using Game Trees, the Minimax algorithm, and the alpha-beta pruning search algorithm. My heuristic used rectangular approximations and distance calculations to evaluate the desirability of a board state. For extra-credit, I made the GUI fully functional and added features such as highlighting legal moves.

  • Organization: UC Berkeley CS61B (Data Structures)
  • Tech: Java
  • Link: Spec
  • Date: April 2020
