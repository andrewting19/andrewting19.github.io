Tools I Actually Use

2026-02-09 · 7 min read ai tooling workflow

no cap no yap AI has written all my code for the past 6 months. no more IDE. no VS Code, no Cursor, no ma nothing. just good ol TUIs and claude code which definitely has no TUI bugs whatsoever. yup i'm not kidding. Coding agents do literally 100% of my coding now. all I do is yap at my computer and code gets written. bada bing bada boom. It's hilarious. It also actually works. It doesn't just work, it fucking flies.

now for the juicy content: here's the unfiltered recap of the tools that I use every day.

How my day actually works

I wake up, open Agentboard. What's agentboard you ask? It's a web based UI wrapper around tmux, and I do all of my work from it. I'll usually have a few Claude Code sessions still running from the night before. I try to work in parallel. Read what Claude wrote, think about it critically with what remains of my slowly atrophying brain, write a prompt, fire it off, switch to the next thing while it works. Waiting for an agent to finish is dead time, and the most efficient use of it is sending another agent a different task. Not scrolling Twitter.

Which is why I immediately open Twitter and start scrolling. I was so chronically online and addicted to doomscrolling Twitter between prompts that I built a Chrome extension I'm quite proud of called Hush Focus. It completely blocks Twitter and YouTube unless I have an active Claude Code or Codex session running. The second an agent finishes and I have nothing running, access gets cut. It also plays a noise everytime a session finishes responding. Now you might think that it's insane and cringe that I had to build software to stop myself from scrolling because I have no self control, and to that I tell you well now all of my dopamine is from wrangling Claude Codes to do my bidding and I'm now a 10x AI engineer TM. I credit my success to this one chrome extension.

The main issue with running many things in parallel is the context switching tax. There's a cost to multi-tasking. If the agents weren't so slow I would iterate on one thing at a time but because it can take 5-10 mins for a single task, I'm forced into this workflow. I'm still getting better at multi-tasking. My reading and comprehension skills are getting faster. But even running just two or three tracks at once is way more productive than the old loop of prompt, wait, stare at terminal, prompt again. And vastly more productive than growing a crop of organic free-range hand-written artisinal human-authored code.

YOLO mode (in a box)

For the last 6 months I've also exclusively run Claude Code and Codex with in yolo mode. Zero permission restrictions. No sandbox. No approval prompts. No nothin'.

This was the first 10x speedup. Manually approving stuff forces you to stare at the terminal waiting for input. This unlocks autonomy and parallel workstreams. Now before you freak out, I was smart about it. This is all inside Docker. My setup is open-source and you can find it here: Claude Ting. It has some key useful features. The agent can only see the mounted project folder. Auth is automatic so no relogging every time. You get the speed of YOLO mode without the risk of Claude deleting your home directory.

My take on YOLO mode: I used to be scared of it too. I came from Cursor where I ran YOLO mode just fine, but something about Claude Code having bash access made me nervous. So I built the container instead of dealing with the anxiety. But honestly? The models are smart enough now that catastrophic mistakes basically don't happen. If you write decent prompts and you're working in a reasonable codebase, the chance of the agent going rogue is close to zero. The container is a safety net I rarely actually need.

I've been running this setup since mid-2025. It was the biggest single unlock for my productivity. Before this, I was clicking "approve" on every file edit like some kind of bureaucrat.

Agent Board

Agent Board is a web UI for tmux. I'm one of the main contributors. It sounds simple - and it is - but it changed how I work.

Every Claude Code session runs in tmux. Agent Board puts them all in a browser layout where I can see what each one is doing. When an agent spins up a sub-agent, Agent Board automatically discovers it because it's just another tmux session. I can watch, I can jump in and take over manually, I can kill it. Total visibility, all the time.

It's honestly better than Ghostty for my use case. Sorry, Ghostty. The scrolling is a little janky because tmux, but the tradeoff is worth it.

The other thing - because it's in the browser, I can use it from my phone. I have Tailscale on my laptop, phone, and VPS. So if I'm walking my dog, getting a haircut, whatever - as long as my laptop is on, I can manage all my agents from my phone. The bottleneck is just screen size and not having a keyboard. This could honestly be its own blog post too.

Agents spawning agents

Claude Code has built-in sub-agents. They're fine. But you can't see what they're doing and you can't take over if they go off track. So I built the Dev Sessions MCP - it's part of the Claude Ting repo.

It's dead simple. Creating a "developer" just starts a tmux session and runs Claude or Codex in it. The MCP gives you two operations: send a message to a developer, and read the last N lines of their output. That's the whole thing.

But what this enables is nuts. My main Claude Code instance becomes a manager. I tell it what I want, it figures out the sub-tasks, spins up developers for each one, monitors them, and brings the results together. And because each developer is just tmux, Agent Board picks them all up automatically. I have eyes on everything.

I use this pattern so much I made slash command skills for it:

  • /dev-control - orchestrate tasks across multiple developers with full visibility via the MCP
  • /orchestrate - similar but using Claude Code's built-in sub-agents
  • /handoff - this one's important. When context fills up (~200k tokens), start a fresh agent with a briefing that only includes what it needs to continue the work

I need to talk about /handoff for a second. Claude Code's built-in compaction is god-awful. I hate it. It throws away context that matters and the agent gets confused. My approach is different - I don't try to compress the existing session. I just start a new one and have the current agent write a briefing with only the relevant details for what comes next. The workflow continues like nothing happened. It works really, really well.

Context management is probably the single hardest part of working with Claude Code right now. The 200k window sounds big until you realize that just reading all the relevant files and docs puts you at 100k before you've even started working. Most of the skills I've built are just different strategies for dealing with this.

I yap at my computer

The second biggest speed unlock after YOLO mode: voice-to-text prompting.

I use NVIDIA's Parakeet speech-to-text models running locally through an open-source app called Handy. Forget SuperWhisper. Forget WhisperFlow. Just download the Parakeet model and run it yourself. It's faster, it's at least as accurate, and it's free.

So yeah. I literally just talk to my computer, the transcription goes into Claude Code, and code comes out. I can explain complex context, constraints, edge cases - all the stuff that's tedious to type but natural to say out loud. It's a weird workflow. I look unhinged. But it's fast.

The stack

Claude Code

The coding agent. Replaced Cursor and it's not close. Better results, faster, and with YOLO mode it's a different experience entirely.

Claude Ting

Docker sandbox for Claude Code. YOLO mode without the fear. Handles auth, mounts your project, keeps your system safe. Also includes the Dev Sessions MCP for agent orchestration.

Agent Board

Web UI for tmux. All your agent sessions in the browser. Auto-discovers sub-agents. Take over any session anytime. I'm a main contributor.

Hush Focus

Chrome extension. Blocks Twitter and YouTube unless you have active coding sessions. Turns out the hardest part of AI coding is not scrolling.

Handy + NVIDIA Parakeet

Local speech-to-text. Free, fast, runs on your machine. The voice-prompting thing sounds dumb until you try it.

Has this saved me time? An insane amount. I'm a solo dev shipping at a pace that would've needed a team not that long ago.

Has it saved me money? No. I pay so much money to Anthropic. I'm pretty sure I'm personally increasing their shareholder value. But the leverage you get is worth every penny.

Everything here is open source. The setup takes an afternoon. The hardest part is letting go - trusting the agent enough to stop hovering over every file edit and just letting it work.
