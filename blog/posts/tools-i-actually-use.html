Tools I Actually Use

2026-02-09 · 7 min read ai tooling workflow

Coding agents have done 100% of my coding work for the past six months. I haven't manually edited a file in an IDE since I switched from Cursor to Claude Code. That sounds like hyperbole. It's not.

Here's what my actual setup looks like — no sponsored tools, no affiliate links, just the stuff I use every day to ship software by talking to my computer.

The Daily Loop

I start the day by firing off tasks to coding agents in parallel. The workflow is simple: write a prompt, send it, switch context to the next task while it works, come back when it's done. Repeat.

This sounds easy but there's a real context-switching tax. The trick is having 2–3 tracks running simultaneously — enough to stay productive during wait times, not so many that you lose track of what's happening. I'm actively getting better at this.

The biggest enemy of this workflow? Twitter. So I built a Chrome extension called Hush Focus that blocks my access to Twitter and YouTube unless I have an active Claude Code or Codex session running. The moment an agent finishes and nothing is actively working, my access gets cut. It's implemented using Claude Code hooks and smart log polling for Codex. Honestly, this could be its own blog post.

Claude Code in a Docker Sandbox

I run Claude Code and Codex exclusively inside a Docker sandbox with zero permission restrictions. No approval prompts, no "are you sure?" dialogs, nothing. Full YOLO mode.

"Isn't that dangerous?"

Here's my take: the models used to be dumb enough to accidentally nuke your filesystem. They're not anymore. With current-generation models, the probability of a catastrophic mistake is near zero — if you're reasonably precise in your prompts. The only thing stopping most people from going full-auto is trust. And I solve the trust problem with a container.

My setup is Claude Ting — an open-source Docker sandbox for Claude Code. The agent can only access the mounted project folder. Auth is handled automatically (no re-login headaches). You get full YOLO-mode speed with none of the risk. I've been running this since mid-2025 and it's the single biggest unlock for my productivity.

YOLO mode + Docker isolation was the first big speed unlock. Voice-to-text prompting was the second.

Agent Board

I manage all my coding sessions through Agent Board — a web-based UI wrapper around tmux. Think of it as your terminal, but in the browser, with every coding agent session visible in a clean layout.

Why this matters: when your agents spin up sub-agents (more on that below), Agent Board automatically discovers them as new tmux sessions. I get full visibility into what every agent is doing at all times. If something goes sideways, I can jump into any session and take over manually. This flow — automated by default, manual takeover when needed — is incredibly powerful.

The other killer feature: because it's browser-based, I can code from my phone. I have Tailscale connecting my laptop, phone, and VPS. As long as my laptop is on — even if I'm walking my dog or getting a haircut — I have full access to all my coding agents from my phone. The only bottleneck is screen size and not having a keyboard.

Orchestrating Agents with Agents

Claude Code's built-in sub-agents work fine, but you can't see what they're doing or take over if they go off the rails. So I built the Dev Sessions MCP — an MCP server that lets Claude Code spawn and manage other Claude Code instances.

Under the hood, it's simple: creating a "developer" just starts a tmux session and runs Claude or Codex in it. The MCP exposes two operations — send a message to a developer, and read the last N lines of their output. That's it.

But this is where it gets interesting. My main Claude Code instance acts like a manager. It breaks down complex tasks, delegates to sub-agents via the MCP, monitors progress, and synthesizes results. And because every sub-agent is just a tmux session, Agent Board picks them up automatically — so I always have eyes on everything.

I use this enough that I built custom skills (slash commands) for common patterns:

  • /orchestrate — break a task into sub-tasks and delegate to built-in sub-agents
  • /dev-control — same thing but using the Dev Sessions MCP for full visibility
  • /handoff — when context gets full (~200k tokens), spin up a fresh agent with only the relevant context from the current session. Basically lossless context continuation without the garbage built-in compaction

The /handoff skill deserves special mention. Claude Code's built-in compaction is awful — it throws away important context and confuses the agent. My approach is to start a new session entirely, with a carefully crafted briefing that carries over only what matters. It works shockingly well.

Voice-to-Text Prompting

The second biggest speed unlock after YOLO mode: I prompt my coding agents with my voice.

I use NVIDIA's Parakeet speech-to-text models running locally through an open-source app called Handy. It's faster and more accurate than SuperWhisper or WhisperFlow, and it's free. Download the model, run it on your machine, done.

So my actual workflow is: I yap at my computer, the transcription gets piped into Claude Code, and code gets written. It's genuinely hilarious. But it's also genuinely faster than typing — especially for complex prompts where you need to explain context, constraints, and desired behavior in detail.

The Stack

Claude Code

The coding agent. Replaced Cursor completely. Better results, faster execution, and with YOLO mode enabled it's not even close.

Claude Ting

My Docker sandbox for running Claude Code in full-auto mode. Open source. Handles auth, mounts your project folder, keeps your system safe.

Agent Board

Web UI for tmux. See all your agent sessions, take over anytime, access from your phone. I'm a main contributor.

Dev Sessions MCP

MCP server for agent-to-agent orchestration. Part of the Claude Ting repo. Lets your main agent spawn and manage sub-agents with full visibility.

Hush Focus

Chrome extension that blocks distracting sites unless you have active agent sessions. Accountability through automation.

Handy + NVIDIA Parakeet

Local speech-to-text. Free, fast, accurate. The voice-prompting unlock is real.

Has this saved me time? An absurd amount. Has it saved me money? Absolutely not — I'm pretty sure I'm Anthropic's biggest customer at this point. But the leverage is insane. I'm a solo developer shipping at a pace that would've required a team two years ago.

The tools are all open source. The setup takes maybe an afternoon. The hardest part is trusting the agent enough to let go of the keyboard.
